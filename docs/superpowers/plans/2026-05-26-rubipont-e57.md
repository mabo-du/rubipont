# rubipont Phase 2 — E57 Read Support

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans.

**Goal:** Add E57 read support to rubipont, enabling conversion from E57 to LAS, LAZ, and PCD.

**Architecture:** Extend `rubipont-core/src/format/` with an `e57` module that implements `PointCloudReader` via the `e57` crate. Single-scan read for MVP (multi-scan E57 reads the first point cloud and warns about additional scans).

**Tech Stack:** `e57` 0.11.12 (already in Cargo.toml), existing rubipont-core pipeline.

---

**Implementation:**

`format/e57.rs`:
```rust
use std::path::Path;
use e57::E57Reader;

use crate::error::{Result, RubipontError};
use crate::layout::{PointChunk, PipelineContext, PointLayout};
use crate::pipeline::{PointCloudReader, PointCloudWriter};

pub fn detect(ext: &str) -> bool {
    ext.eq_ignore_ascii_case("e57")
}

pub struct E57ReaderImpl {
    reader: E57Reader<std::io::BufReader<std::fs::File>>,
    pc_index: usize,         // which point cloud we're reading
    total_points: u64,
    metadata: PipelineContext,
    point_size: usize,
    exhausted: bool,
}

impl E57ReaderImpl {
    pub fn new(path: &Path) -> Result<Self> {
        let reader = E57Reader::from_file(path).map_err(|e| RubipontError::ParseError {
            format: "E57".into(),
            offset: 0,
            detail: e.to_string(),
        })?;

        let pointclouds = reader.pointclouds();
        if pointclouds.is_empty() {
            return Err(RubipontError::ParseError {
                format: "E57".into(),
                offset: 0,
                detail: "No point clouds found in E57 file".into(),
            });
        }

        if pointclouds.len() > 1 {
            // For Phase 2, we read the first point cloud and warn about others
            eprintln!("Warning: E57 contains {} point clouds; reading first one only", pointclouds.len());
        }

        let pc = &pointclouds[0];
        let total_points = pc.records;

        let mut metadata = PipelineContext::default();

        // CRS metadata from E57 XML
        if let Some(crs) = reader.coordinate_metadata() {
            // Store in metadata — will need PipelineContext::crs_wkt field
            // For now, we just have coordinate_scale and coordinate_offset
        }

        // E57 uses f64 Cartesian coordinates natively (no integer scaling)
        // Point size in our internal format: 3×f64 + u16 = 26 bytes
        let point_size = 26usize;

        Ok(Self {
            reader,
            pc_index: 0,
            total_points,
            metadata,
            point_size,
            exhausted: false,
        })
    }
}

impl PointCloudReader for E57ReaderImpl {
    fn read_chunk(&mut self) -> Result<Option<PointChunk>> {
        if self.exhausted {
            return Ok(None);
        }

        // Get the first point cloud
        let pointclouds = self.reader.pointclouds();
        if self.pc_index >= pointclouds.len() {
            self.exhausted = true;
            return Ok(None);
        }

        let pc = &pointclouds[self.pc_index].clone();

        // Create point cloud iterator
        let mut iter = self.reader.pointcloud_simple(pc).map_err(|e| {
            RubipontError::ParseError {
                format: "E57".into(),
                offset: 0,
                detail: e.to_string(),
            }
        })?;

        let mut data = Vec::with_capacity(4096 * 26);
        let mut count = 0usize;

        for pt_result in iter.by_ref().take(4096) {
            let pt = pt_result.map_err(|e| RubipontError::ParseError {
                format: "E57".into(),
                offset: count as u64,
                detail: e.to_string(),
            })?;

            // Extract Cartesian coordinates
            let (x, y, z) = match pt.cartesian {
                e57::CartesianCoordinate::Valid { x, y, z } => (x, y, z),
                e57::CartesianCoordinate::Direction { x, y, z } => (x, y, z),
                e57::CartesianCoordinate::Invalid => (f64::NAN, f64::NAN, f64::NAN),
            };

            // Extract intensity (normalize from 0-1 to u16 0-65535)
            let intensity = match pt.intensity {
                Some(val) => (val.clamp(0.0, 1.0) * 65535.0) as u16,
                None => 0,
            };

            data.extend_from_slice(&x.to_le_bytes());
            data.extend_from_slice(&y.to_le_bytes());
            data.extend_from_slice(&z.to_le_bytes());
            data.extend_from_slice(&intensity.to_le_bytes());
            count += 1;
        }

        self.pc_index += 1; // For multi-scan: advance to next
        if count == 0 {
            self.exhausted = true;
            return Ok(None);
        }

        Ok(Some(PointChunk { data, len: count }))
    }

    fn layout(&self) -> &PointLayout {
        // Return a static layout — point_size known at construction
        // For now, returning a cached size
        use std::sync::OnceLock;
        static LAYOUT: OnceLock<PointLayout> = OnceLock::new();
        LAYOUT.get_or_init(|| PointLayout {
            point_size: 26,
            num_points: self.total_points,
            has_integer_coords: false,
        })
    }

    fn metadata(&self) -> &PipelineContext {
        &self.metadata
    }
}
```

Note: The above has an issue with `layout()` — it uses `OnceLock` but that requires `self` to be accessible at the time of definition. A better approach is to store `PointLayout` as a field on the struct. The subagent should store it as a field.

**Implementation hint for the subagent:** The `PointCloudReader` trait returns `&PointLayout` from `layout()`. The simplest approach is to store a `PointLayout` as a field on the struct, initialized in `new()`.

For the test file, E57 test files are harder to generate programmatically (no simple builder like LAS). Options:
1. Find a small E57 file in the e57 crate's test data
2. Write a points CSV and convert to E57 using e57 writer programmatically in the test

Option 3 is most robust. The test can create a small E57 file in-memory using the e57 writer, then read it back.
